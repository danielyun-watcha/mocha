# EDA Answer Templates

Lead Opus 가 사용자 질문 유형에 맞춰 4개 템플릿 중 하나를 골라 답변에 사용. 모든 템플릿에 **"인사이트 / 시사점" 컬럼 또는 섹션 필수** (Toss PANDA 패턴 — 단순한 답이라도 인사이트 첨부).

## 4 템플릿

| # | 파일 | Use when | 분량 | 예상 시간 |
|---|---|---|---|---|
| 1 | `01_light_memo.md` | TOP N · 분포 · 평균 · 카운트 (single-shot) | 30-50줄 | ~15-30초 |
| 2 | `02_full_eda.md` | 전반 데이터 분석 / EDA 보고서 / "특성 파악" | 150-300줄 | 2-5분 |
| 3 | `03_ab_test.md` | A/B test 사후 분석 / 실험 KPI 비교 | 200-400줄 | 3-7분 |
| 4 | `04_analysis_report.md` | 분석 노트 / 미완료 정리 / 1-3개 데이터 mid-size 분석 | 100-200줄 | 1-3분 |

## 라우팅 가이드 (Lead 가 자율 판단)

```
질문 → 분류:

1. "TOP N", "분포", "평균", "카운트", "X별 Y 수치", 단순 시각화
   → Template 1 (Light Memo)

2. "EDA", "전반 분석", "데이터 특성", "어떻게 생겼어", "전체 분석"
   → Template 2 (Full EDA)

3. "A/B test", "실험 결과", "모델 A vs B 비교", "온라인 KPI"
   → Template 3 (A/B Test)

4. "이 데이터로 X 분석해줘", "다음 질문들에 답해줘", 1-3개 데이터 소스
   → Template 4 (Analysis Report)

5. 명확하지 않으면 → Lead 가 1회 묻거나 가벼운 1 (Light) 부터 시도
```

## 공통 원칙

**모든 템플릿에 적용**:

- **PANDA 답변 구조**: 결과(표) + 집계 기준 + 인사이트
- **단순한 답에도 인사이트 1-2개 포함** — 표만 던지지 말 것
- **표는 가능하면 "수치 + 시사점 / 해석" 컬럼 동반**
- **시각화는 inline `![](/eda-files/X.png)`** — 한국어 폰트 적용
- **차트 1개당 파일 1개** (subplot 합치지 말 것)
- **Watcha 테마 색상**: TOP 3 강조 `#d97757 / #6a9bcc / #788c5d`, 비강조 `#e8e6dc`
- **시간대는 KST 보정 필수** (unix ts 는 UTC + 9h)
- **데이터 함정 항상 검증**: 필터링 인공물 / 스케일 오해 / 헤비유저 영향 / Cold start

**금지**:

- 표 수치 그대로 다시 적기 (인사이트는 "so what")
- "흥미롭다", "다양한 패턴" 같은 hedge
- 사용자 묻지 않은 곁가지 권장 (단, 데이터 함정 알림은 OK)
- Light Memo 에 Full EDA 분량 (사용자가 짧은 답 원함)
- Full EDA 에 Light Memo 깊이 (사용자가 깊은 답 원함)

## 사용법

Lead 가 답변 작성 전:

1. 사용자 질문 → 위 라우팅 가이드로 템플릿 선택
2. `Read` 해당 템플릿 파일 (예: `Read templates/02_full_eda.md`)
3. 템플릿의 `{placeholder}` 부분을 실제 분석 결과로 채움
4. 사용자에게 답변
