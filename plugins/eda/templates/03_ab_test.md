# A/B Test 사후 분석 템플릿

**Use when**: "A/B test 결과 분석", "실험 사후 분석", "모델 X vs Y 비교", "온라인 KPI 검증". 매출/전환/다양성 등 비즈니스 메트릭 평가.

**기준 자료**: Rec Adult Diff A/B test (HSTU vs DIFF) 노션 페이지의 검증된 구조.

---

- **{핵심 지표 2}**: 기존 대비 약 {%} 변화 ({값}, {p-value})

## 테스트 목표 (4분류 — Watcha 표준)

- **Success + Deterioration** (성공 또는 악화 양 방향 봐야 할 핵심 KPI)
  - {ARPU, 1인당 매출 등}
- **Success + Guardrail** (성공 + 부작용 막아야 할)
  - {1인당 클릭 수, 구매 전환율 등}
- **Guardrail** (악화하면 안 되는)
  - {ARPPU, 재구매율, 리텐션 등}

## 작성 원칙

- **모든 메트릭은 차이 + p-value 동반** — 통계적 유의미성 명시
- **세그먼트별 분석** — 라이트/헤비/그룹별로 쪼개기. 전체 평균만 보면 헤비유저 영향 못 잡음
- **DID 또는 baseline 비교** — 실험 전후 또는 다른 그룹과 차이
- **이상치/Top1 시뮬레이션** — 한 명의 영향 분리 검증
- **이전 실험과 비교** — 같은 도메인 이전 실험 결과 옆에 놓기
- **한계 명시** — 기간 짧음 / 표본 / 도메인 특수성
