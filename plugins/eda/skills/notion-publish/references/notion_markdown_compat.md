# Notion 마크다운 호환 가이드

`notion-create-pages` MCP가 받는 마크다운 작성 시 주의 사항.

## 잘 동작하는 것

- **헤딩**: `# H1` ~ `### H3` (Notion은 H3까지 지원 — H4+ 는 본문으로 표시)
- **표**: 마크다운 파이프 표 (`| col | val |`) — 자동 변환됨
- **목록**: `-` / `*` / `1.` 모두 OK
- **인라인 코드/굵게/기울임**: 표준 마크다운대로
- **링크**: `[text](url)` — 외부 URL은 클릭 가능
- **이모지**: 텍스트 안의 이모지(`💡`, `📊`) 그대로 표시
- **인용**: `> ...`
- **수평선**: `---`

## 주의가 필요한 것

| 요소 | 동작 | 우회법 |
|---|---|---|
| 로컬 이미지 `![](path.png)` | Notion에서 깨짐 | 외부 호스팅 (S3, Imgur) 후 URL 교체 |
| H4 이상 헤딩 | 본문으로 표시됨 | H3까지만 사용 |
| 복잡한 표 (병합 셀, 줄바꿈 내포) | 부분 손상 | 단순 표로 분할 |
| 코드 블록 fence 언어 | `python` / `bash` 등 표준만 정확 인식 | 표준 식별자만 사용 |
| HTML 직접 삽입 (`<details>` 등) | 무시되거나 raw 텍스트 | Notion 블록 사용 권장 |
| 수식 (`$...$`) | Notion 수식 블록으로 변환 안 됨 | `inline math` 블록 별도 |

## EDA 리포트 특수 케이스

PANDA 5단 구조 (`# 헤더`, `## 📅 결과`, `## 📊 집계 기준`, `## 💡 주요 인사이트`, `## 📎 Appendix`)는 모두 H1~H2 + 마크다운 표로 구성되어 Notion에 잘 들어간다.

**남는 문제**: 이미지 임베드 (F1~F6 PNG). 두 가지 옵션:
1. **alt만 남기기**: `![데이터 개요](path.png)` → `_(데이터 개요 — 별도 첨부)_` 로 사전 치환
2. **외부 호스팅**: S3 등에 업로드 후 URL로 교체 — Notion이 잘 렌더링함

`notion-publish` 기본 동작은 (1) — 사용자에게 한계 안내. (2)는 사용자가 외부 호스팅 마친 후 `--upload-images` 모드를 별도로 호출하는 식 (현재 미구현).
