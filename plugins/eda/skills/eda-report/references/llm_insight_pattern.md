# LLM Insight Writing — 압축 패턴 (PANDA-style)

이 문서는 EDA 리포트의 인사이트 섹션을 **오케스트레이터(Claude)** 가 작성할 때 따라야 할 압축 패턴.

> ⚠️ **가장 중요한 원칙**: 사용자가 묻지 않은 것은 답하지 않는다.

## 🚨 모든 답변에 강제되는 3원칙

### 1. 데이터 / 기간 / 사용처 명시 (필수)

모든 답변에 다음을 **반드시** 포함:
- **데이터**: 사용한 파일 경로 (예: `graph_modeling/builtin/train.ftr`)
- **기간**: 데이터 수집 기간 + **KST 보정 여부**
- **사용처**: 결합한 보조 데이터 (예: `content_tag_edges.pkl` + `tag_id_to_name.pkl`)
- **계산 기준**: 핵심 metric 정의 (Lift, Cosine, MEH 비율 등)

이게 빠지면 사용자가 답변 신뢰성 판단 불가.

### 2. ML 용어 풀어쓰기 (PM/Infra 친화)

ML 전공자만 이해할 용어를 **무조건 한 줄 정의**로 풀어 설명:

| ML 용어 | 풀어쓰기 예시 |
|---|---|
| Lift X.XX | "정상 시청 점유율 대비 MEH 점유율 비율. Lift 1.5 = 시청량 대비 MEH를 1.5배 더 받음" |
| Cosine similarity | "두 콘텐츠가 가진 태그(장르) 구성이 얼마나 비슷한가. 0=전혀 다름, 1=완전 같음" |
| Sparsity | "유저-콘텐츠 매트릭스가 얼마나 비어 있는가. 99%면 100칸 중 1칸만 채워짐" |
| Gini 계수 | "콘텐츠 인기 불균등 정도. 0=완전 균등, 1=한 콘텐츠가 다 차지" |
| GNN / CF | "그래프 기반 / 협업 필터링 추천 — 유저·콘텐츠 관계를 학습" |
| Embedding | "콘텐츠/유저를 숫자 벡터로 표현해 유사도 계산 가능하게 함" |
| K-core | "최소 N번 이상 활동한 유저/콘텐츠만 남기는 전처리 필터" |
| Power-law | "상위 소수가 전체의 대부분을 차지하는 분포 (영화 인기 분포가 대표적)" |

핵심 원칙: 표/차트에 ML 용어 등장 시 **표 바로 아래** 또는 **glossary** 형태로 정의.

### 3. 콘텐츠는 실제 제목으로 (가능 시)

`content_key` (예: `1:82543`) 같은 raw ID는 PM/비즈니스 팀이 모름. 실제 영화 제목으로 변환:

**조회 가능한 source**:
- BigQuery: `gretel.production_us.mars_content_view_summary_*` 의 `content_title` 필드
- BigQuery: `gretel.frograms_us.nuvus_properties` 의 `title`
- 우리 datasource: `remy.ds.bq.content.MarsContent`

**조회 가능 시 표시 형식**:
```
| 콘텐츠 | MEH | ... |
|---|---|---|
| 영화 「범죄도시」 | 1,179 | ... |
```

**조회 불가 시 (archive 데이터만 사용)**:
```
| 콘텐츠 | MEH | ... |
|---|---|---|
| Movie #82543 (드라마) | 1,179 | ... |
```

또는 답변 끝에 안내:
> ⚠ 영화 제목은 BigQuery `mars_content_view_summary` 조회 필요. 현재 답변은 archive 데이터만 사용 — content_key를 실제 제목으로 변환하려면 사용자가 권한 부여 또는 별도 매핑 제공해야 함.

## 답변 압축 5원칙

### 1. 사용자 질문에 직접 답만
- 질문 = "장르 선호" → 장르 표 + 인사이트만. 시간대 / 유저 segment / 모델링 권장 등 **묻지 않은 곁가지 X**.
- 모델링 권장은 사용자가 모델링 개선을 명시했을 때만.

### 2. 표는 1-2개, 짧게
- 핵심 표 1개로 답할 수 있으면 1개만.
- 행 7-10개, 컬럼 3-4개 이내.

### 3. 인사이트 3-5개, 각각 컴팩트
- ✓ 한 줄 결론 (제목)
- 2-3개 bullet (수치 + 짧은 해석)
- bullet 한 줄 ≤ 1문장

### 4. 사과 / 한계 / 옵션은 마지막 한 줄로만
- "데이터에 X 없음 → Y 데이터 추가 필요" — 한 줄
- 추가 분석 옵션 — 1-2개 키워드만

### 5. 메타 정보 제일 짧게
- 데이터 / 기간 / 행 수 — 한 줄로

## 표준 답변 구조 (PANDA)

```markdown
🐼 [질문 한 줄 요약]

📅 **[핵심 표 1]**
| col | col | col |
|---|---|---|
...

📅 **[핵심 표 2 — 필요시만]**
...

📊 **집계 기준**
- 데이터: ... · 기간: ... · 행 수: ...
- 기준 / 해석: 한 줄

---

💡 **주요 인사이트**

✓ **[결론 1 — 한 줄, 굵게]**
- 수치 부연 1
- 수치 부연 2
- 해석 1줄

✓ **[결론 2]**
- ...

✓ **[결론 3]**
- ...

(끝 — 권장사항 / 추가 데이터 / 옵션은 별도 안내 X. 사용자가 추가 질문하면 그때 답.)
```

## ❌ 피해야 할 패턴

### 1. 4관점 라벨 텍스트 (관찰/비즈니스/유저/모델링)
- 사용자가 명시적으로 4관점 요청 안 했으면 절대 사용 X
- 답변이 무거워지고 사용자 질문에서 멀어짐

### 2. 곁가지 표·차트 다 보여주기
- 사용자 질문과 무관한 표 (예: 장르 묻는데 유저 segment 표) **삭제**
- 표 한 개에 모든 통계 우겨넣지 말 것

### 3. "추가 분석 권장" 5-10개
- 1-2개 키워드 옵션만. 길게 권장 X.

### 4. 동일 수치 여러 섹션 반복
- 핵심 요약 + 표 + 인사이트가 같은 숫자 반복하면 verbose
- 표는 raw, 인사이트는 **표에 없는 의미** 추가

### 5. 시간대 UTC 그대로 사용
- timestamp는 무조건 KST(+9h) 보정 후 해석
- "점심대" 같은 결론은 반드시 **변환 후** 도달

## ✅ 권장 패턴 예시

### 질문: "왓챠 사용자가 좋아하는 장르?"

```markdown
🐼 왓챠 사용자 선호 장르

📅 **TOP 7 장르**
| 장르 | 시청 점유 |
|---|---|
| 드라마 | 48.3% |
| 애니메이션 | 33.8% |
| 로맨스 | 28.5% |
| ... | ... |

📅 **Movie vs Series — 패턴 정반대**
| 순위 | 🎞️ Movie | 📺 Series |
|---|---|---|
| 1 | 드라마 62% | 애니메이션 52% |
| 2 | 로맨스 27% | 드라마 38% |
| 3 | 스릴러 19% | TV드라마 37% |

📊 **집계 기준**
- 데이터: graph_modeling/builtin · 2026-01-13 ~ 2026-05-12 (KST 보정)
- 기준: 시청 1건 = 1 view · 콘텐츠 multi-tag (Genre + Genre::Main)

---

💡 **주요 인사이트**

✓ **드라마/애니/로맨스 3대 장르 시장 지배**
- 4위 코미디(22%)와 격차 큼
- 한 콘텐츠 multi-tag — 합 > 100% 정상

✓ **Movie ≠ Series — 완전히 다른 콘텐츠**
- 영화 시청 = 드라마 62%, 시리즈 시청 = 애니 52%
- 동일 유저도 모드별 소비 분리

✓ **시간대 무관 안정적**
- 24시간 동안 드라마/애니/로맨스 TOP3 동일
```

위 답변 = **35줄 이내** + 사용자 질문에 직접 답.

## 자가 체크 (작성 전 / 작성 후)

- [ ] 사용자가 명시적으로 묻지 않은 정보 있는가? → 삭제
- [ ] 표 / 차트가 답변에 필수적인가? → 아니면 삭제
- [ ] 인사이트 3-5개? (10개 이상이면 압축)
- [ ] bullet 한 줄 ≤ 1문장?
- [ ] 동일 수치 반복 없는가?
- [ ] 시간대 데이터 KST 보정 했는가?
- [ ] 사과 / 한계 / 옵션 마지막 한 줄?

작성 후 답변 길이 — **목표: 30~50줄 이내** (full 리포트는 100줄 이내).

## 모드별 답변 길이 가이드

| 모드 | 길이 | 표 | 인사이트 |
|---|---|---|---|
| Q&A (자연어 질문) | 20-40줄 | 1-2개 | 3-5개 |
| 풀 리포트 | 80-150줄 | 3-5개 | 5-7개 |
| Appendix (전체 상세) | 무제한 | 모든 분석 | deterministic 결과 |
